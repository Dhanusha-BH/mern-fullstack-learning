Client-Server Architecture

This webpage explains the concept of client-server architecture and traces what happens when a user opens a website in a browser.

What is Client?

A client is a device or application that requests service or resources from a server. Example of clients include web browsers like Google chrome, Mozilla Firefox, and mobile applications.

What is Server?

A server is a powerful computer or system that stores,processes,and provides data or services to clients. It listens to clients request and sends appropriate response.

Client-Server Architecture

client-server architecture is a computing model where multiple clients communicatewith a central servr. The client sends a request, and the server process the request and sends back a response.

What Happens When You Open a Website?

  1. The user enters a website URL (for example, www.google.com)in the browser.
  2. The browser send a request to the DNS server to get the IP address of the website.
  3. After receiving the IP address, the browser sends an HTTP/HTTPS request to the web server.
  4. The server sends a response containing HTML, CSS, and Javascript files back to the cilent.
  5. The browser renders the received files and displays the webpage to the user.

Conclusion

Client-server architecture is the backbone of modern web applications. Understanding this flow helps in learning full stack web development and how frontend and backend system interact.
